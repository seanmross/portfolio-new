bloccit

overview

Sports, music, technology, food, travel and beyond - for all topics of conversation in our everyday lives, we have convenient sites such as Reddit that aggregate this information for us. I have created Bloccit, a Ruby on Rails application, to provides a similar experience. It is a forum space for users, and depending on their authorization level, users have the ability to create an account, create posts, and comment on/favorite other users posts.

see on github

features
  • sessions were created to keep track of a users state, keeping them logged in to the site
  • user specific roles - admin, member, and guest with authorization rules for each level were created
  • users have the ability to upvote/downvote or favorite posts
  • users have a profile page where they can view their posts and favorites
  • posts are filtered by a time-decay algorithm and voting score
ruby gems
  • bcrypt - for sensitive user information encryption
  • bootstrap - css framework
  • figaro - to securely store Sendgrid and Devise credentials as environment variables
  • rspec - testing framework for test driven development (tdd)

bloc-jams

overview

Music plays a huge part in my life. I'm frequently using apps like Soundcloud and Spotify throughout the day. For my first app, I wanted to make a product I would love to use. I created Bloc-Jams, which is a music player web-app similar to Spotify. Bloc-Jams was originally created using DOM scripting and jQuery library but later expanded using AngularJS for scalability. My goal in creating Bloc-Jams was to learn how to configure Angular routing and states, implement controllers for my app's views, create a service that handles song playback, and write a directive that controls song and volume sliders.

see on github

features
  • the app features view templates for a landing page, album collection page, individual album page, and a player control bar
  • MVC architecture was utilized to dynamically build the views using controllers
  • song playback was handled by injecting a service to share the song playback information across all components of the app
  • directives were used to bind custom user actions to angular functionality, such as adjusting the playback time or volume

wikibloc

overview

It’s everyone’s quick and easy go-to source for information, that first Google search result you’re looking for: Wikipedia. Wikis are the perfect way to collaborate on community sourced content. With Wikibloc, I have created an app that allows users to create their own wikis and share them with collaborators.

see on github

features
  • wikibloc offers different access levels by user role: admin, standard (free), or premium (subscription)
  • standard users can create, read, update, and delete public wikis
  • users can upgrade their account from standard to a paid account
  • premium users have the ability to create private wikis
  • wikis are editable using markdown syntax
  • premium users can add or remove collaborators from their private wiki pages
ruby gems
  • devise - user authentication and account information
  • pundit - role authorization
  • stripe - user account upgrades
  • rspec - test driven development
  • faker - seeding data
